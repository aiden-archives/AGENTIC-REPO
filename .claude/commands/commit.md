# Create Commit

Create a conventional commit with staged changes.

## Steps
1. Run `git status` to see changes
2. Review the diff of staged changes
3. Generate a commit message following conventional commits
4. Create the commit

## Commit Types
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation
- `style:` - Formatting
- `refactor:` - Code restructuring
- `test:` - Adding tests
- `chore:` - Maintenance
